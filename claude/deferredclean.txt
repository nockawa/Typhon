‚óŹ Confirmed Analysis

  Yes, the problem is confirmed. Here's the exact flow:

  1. Cleanup only happens in CommitComponent (line 959) which is called from
   the commit loop
  2. The commit loop only iterates over _componentInfos.Values (line 1490)
  and then componentInfo.CompRevInfoCache.Keys (line 1494)
  3. CompRevInfoCache only contains entries for components this transaction 
  actually touched

  So if T1 (the tail/oldest transaction) never accessed entity E, it has no
  entry in any CompRevInfoCache, and CleanUpUnusedEntries is never called
  for E's revisions.

  The Severity

  This is a memory/storage leak. Old revisions accumulate for any entity
  that:
  - Was created/updated while a long-running transaction was active
  - Was never accessed by that long-running transaction

  In workloads with:
  - Long-running read transactions (analytics, reports)
  - High write activity on other entities

  ...you'd see unbounded growth of revision chains.

  ---
  Proposed Solutions

  Solution 1: Deferred Cleanup Queue (Recommended)

  Idea: Track entities that need cleanup but couldn't be cleaned because of
  older transactions.

  // In DatabaseEngine or TransactionChain
  ConcurrentQueue<(ComponentTable table, long pk, long minTickForCleanup)>
  _pendingCleanups;

  Flow:
  1. When T2 commits and can't cleanup (because T1 is older), enqueue
  (table, pk, T1.TransactionTick)
  2. When T1 commits, process the queue: for each entry where
  minTickForCleanup <= T1.TransactionTick, run cleanup
  3. Or: Have a background thread periodically process the queue when
  minTickForCleanup < TransactionChain.MinTick

  Pros:
  - Cleanup is eventually done
  - Minimal overhead during commit
  - Works with existing MVCC semantics

  Cons:
  - Adds a data structure to maintain
  - Slight delay in cleanup (not immediate)

  ---
  Solution 2: Global Cleanup Pass on Tail Commit

  Idea: When the tail transaction commits, do a full scan of all
  dirty/modified entities since that transaction started.

  // In Transaction.Commit(), when isTail:
  foreach (var componentTable in _dbe.AllComponentTables)
  {
      // Iterate through entities modified between this.TransactionTick and 
  nextMinTick
      CleanupRevisionsInRange(componentTable, this.TransactionTick,
  nextMinTick);
  }

  Implementation requires:
  - A way to efficiently find "entities modified in time range" (perhaps a
  per-table modification log or timestamp index)

  Pros:
  - Cleanup happens immediately when the blocking transaction completes
  - No pending queue to manage

  Cons:
  - Could be expensive if many entities were modified
  - Requires additional tracking of modified entities

  ---
  Solution 3: Revision Chain Lazy Cleanup

  Idea: Clean up old revisions lazily when any transaction accesses an
  entity.

  // In GetCompRevInfoFromIndex(), after finding the revision:
  if (canCleanup && hasOldRevisions)
  {
      CleanUpUnusedEntries(info, ref compRevInfo, accessor, currentMinTick);
  }

  Pros:
  - No additional tracking
  - Cleanup happens naturally during normal access patterns
  - "Hot" entities stay clean, "cold" entities don't matter

  Cons:
  - Adds latency to read operations
  - Never-accessed entities never get cleaned up (storage leak for abandoned
   data)

  ---
  Solution 4: Periodic Background Cleanup

  Idea: A background thread/task that periodically scans all revision chains
   and cleans up old revisions.

  // Background task
  while (running)
  {
      var minTick = _transactionChain.MinTick;
      foreach (var table in _allComponentTables)
      {
          foreach (var pk in table.PrimaryKeyIndex.AllKeys())
          {
              CleanupRevisionsOlderThan(table, pk, minTick);
          }
      }
      await Task.Delay(cleanupInterval);
  }

  Pros:
  - Completely decoupled from transaction commit path
  - Doesn't add latency to normal operations
  - Can be tuned (frequency, batch size)

  Cons:
  - Background thread overhead
  - Delayed cleanup
  - Full scan can be expensive

  ---
  Solution 5: Hybrid - Queue + Lazy (My Recommendation)

  Combine Solutions 1 and 3:

  1. At commit time: When a transaction can't cleanup because older
  transactions exist, enqueue the entity for deferred cleanup
  2. At read time: Check if the revision chain has stale entries and clean
  opportunistically
  3. Queue processing: When the tail commits, process any queued cleanups
  for that tick range

  This gives you:
  - Immediate cleanup when possible (lazy on read)
  - Guaranteed eventual cleanup (queue processing)
  - Minimal overhead (only tracks entities that actually need deferred
  cleanup)

  1. DeferredCleanupManager Class

  public class DeferredCleanupManager
  {
      private readonly SortedDictionary<long, List<(ComponentTable Table,
  long PrimaryKey)>> _pendingCleanups = new();
      private readonly Dictionary<(ComponentTable, long), long>
  _entityToBlockingTick = new();
      private readonly AccessControl _lock = new();

      /// <summary>
      /// Enqueue an entity for deferred cleanup. Called when a transaction 
  commits
      /// but can't cleanup because older transactions exist.
      /// </summary>
      /// <param name="blockingTick">The tick of the oldest transaction 
  blocking cleanup</param>
      /// <param name="table">The component table</param>
      /// <param name="pk">The primary key of the entity</param>
      public void Enqueue(long blockingTick, ComponentTable table, long pk)
      {
          _lock.EnterExclusiveAccess();
          try
          {
              var key = (table, pk);

              // Check if this entity is already queued
              if (_entityToBlockingTick.TryGetValue(key, out var
  existingTick))
              {
                  // Already queued - keep the OLDEST blocking tick
                  // (cleanup should happen when the oldest blocker 
  completes)
                  if (blockingTick >= existingTick)
                      return; // Already waiting for an older transaction

                  // New blocking tick is older - remove from old list, add 
  to new
                  RemoveFromList(existingTick, table, pk);
              }

              // Add to the appropriate tick bucket
              if (!_pendingCleanups.TryGetValue(blockingTick, out var list))
              {
                  list = new List<(ComponentTable, long)>();
                  _pendingCleanups[blockingTick] = list;
              }
              list.Add((table, pk));
              _entityToBlockingTick[key] = blockingTick;
          }
          finally
          {
              _lock.ExitExclusiveAccess();
          }
      }

      /// <summary>
      /// Process all deferred cleanups that are no longer blocked.
      /// Called when a transaction commits and is (or becomes) the tail.
      /// </summary>
      /// <param name="completedTick">The tick of the transaction that just 
  completed</param>
      /// <param name="nextMinTick">The tick of the next oldest transaction 
  (or current time if none)</param>
      /// <param name="changeSet">ChangeSet for tracking 
  modifications</param>
      /// <returns>Number of entities cleaned up</returns>
      public int ProcessDeferredCleanups(long completedTick, long 
  nextMinTick, ChangeSet changeSet)
      {
          List<(ComponentTable Table, long PrimaryKey)> toCleanup = new();

          _lock.EnterExclusiveAccess();
          try
          {
              // Collect all entries where blockingTick <= completedTick
              var ticksToRemove = new List<long>();

              foreach (var kvp in _pendingCleanups)
              {
                  if (kvp.Key > completedTick)
                      break; // Sorted order - no more relevant entries

                  toCleanup.AddRange(kvp.Value);
                  ticksToRemove.Add(kvp.Key);

                  // Remove from reverse lookup
                  foreach (var (table, pk) in kvp.Value)
                  {
                      _entityToBlockingTick.Remove((table, pk));
                  }
              }

              // Remove processed tick buckets
              foreach (var tick in ticksToRemove)
              {
                  _pendingCleanups.Remove(tick);
              }
          }
          finally
          {
              _lock.ExitExclusiveAccess();
          }

          // Perform cleanup outside the lock
          int cleanedCount = 0;
          foreach (var (table, pk) in toCleanup)
          {
              if (CleanupEntityRevisions(table, pk, nextMinTick, changeSet))
                  cleanedCount++;
          }

          return cleanedCount;
      }

      private bool CleanupEntityRevisions(ComponentTable table, long pk, 
  long nextMinTick, ChangeSet changeSet)
      {
          // Get the revision chain for this entity
          using var indexAccessor =
  table.DefaultIndexSegment.CreateChunkAccessor(changeSet);
          if (!table.PrimaryKeyIndex.TryGet(pk, out var compRevFirstChunkId,
   indexAccessor))
              return false; // Entity no longer exists

          using var revTableAccessor =
  table.CompRevTableSegment.CreateChunkAccessor(changeSet);

          // Perform cleanup similar to CleanUpUnusedEntries
          // ... (reuse existing cleanup logic)

          return true;
      }

      private void RemoveFromList(long tick, ComponentTable table, long pk)
      {
          if (_pendingCleanups.TryGetValue(tick, out var list))
          {
              list.Remove((table, pk));
              if (list.Count == 0)
                  _pendingCleanups.Remove(tick);
          }
      }
  }
